## What Was Built This Session

### Core App — `voice.py` (618 lines)
Push-to-talk voice-to-text Windows system tray app. Records speech via microphone, transcribes locally with faster-whisper, and pastes into the active window (any app — Claude Code, ChatGPT, Google Chat, email, etc.).

- **Two modes**: Dictation (Ctrl+Space, light cleanup) and Command (Ctrl+Shift+Period, full cleanup with filler removal, code vocab, optional LLM polish)
- **Hold-to-talk and Toggle mode**: Hold key while speaking, or press once and VAD auto-stops when silence detected
- **System tray icon**: Color-coded (gray=loading, green=ready, red=recording, orange=transcribing)
- **Sound effects**: Pleasant chime .wav files for start/stop/success/error (generated by `generate_sounds.py`)
- **Right-click tray menu**: Toggle all features on/off without editing files
- **Correction mode**: Ctrl+Shift+Z undoes last paste and re-records
- **Wake word**: "Hey Koda" — uses dedicated tiny Whisper model with initial_prompt hint for detection
- **LLM prompt polishing**: Via Ollama (phi3:mini) — cleans up rambling speech into clear instructions
- **Noise reduction**: Optional, via noisereduce library
- **VAD auto-stop**: Silero VAD or energy-based fallback, configurable silence timeout

### Config System — `config.py` + `config.json`
JSON-based config with sensible defaults, deep merge, auto-generation on first run. All settings accessible from tray menu or by editing the file.

### Text Processing — `text_processing.py`
Post-transcription pipeline:
- Filler word removal ("um", "uh", "you know", "basically", etc.)
- Smart capitalization (sentence starts, standalone "i")
- Code vocabulary expansion ("open paren" → `(`, "semicolon" → `;`, etc.)
- Case formatting ("camel case user name" → `userName`, "snake case get data" → `get_data`)

### Interactive Installer — `configure.py` + `configure.bat`
8-step setup wizard:
1. Microphone selection with test and playback
2. Hotkey picker (Ctrl+Space, Ctrl+Period, F-keys, etc.)
3. Recording mode (hold vs toggle)
4. Whisper model size
5. Language
6. Preferences (sounds, filler removal, noise reduction, auto-start)

## Decisions Made

1. **Named "Koda"** — Play on "code" since built in Claude Code. Means "friend/companion." Research confirmed no direct competitors in voice-to-text space. "Zuri" was runner-up but lacked the code wordplay hook.

2. **Wake word = "Hey Koda"** — 2 syllables + "Hey" prefix = 6 phonemes (minimum for reliable wake word detection per Picovoice/Sensory research). Using Whisper tiny model with `initial_prompt="Hey Koda"` instead of a dedicated wake word engine because openwakeword/tflite doesn't work on Python 3.14.

3. **Local LLM via Ollama** — User explicitly wanted no API costs. Chose phi3:mini (2.3GB) as default. Ollama is free, runs locally. Users install separately and pull the model.

4. **Ctrl+Space as default hotkey** — User complained F9/F10 are at the top of the keyboard and would get complaints. Offered ergonomic combos. Ctrl+Space (dictation) and Ctrl+Shift+Period (command) are the defaults.

5. **Notifications disabled by default** — User said the preview popup on the right is redundant since text appears on screen. Turned off.

6. **System beeps replaced with .wav chimes** — User said the winsound.Beep() tones were "not tickling me the right way." Generated pleasant sine-wave chimes instead.

7. **Two modes (Dictation vs Command)** — Dictation = light cleanup (fillers + capitalize). Command = full pipeline (fillers + code vocab + case formatting + optional LLM polish). Different hotkeys for each.

8. **Streaming transcription deferred to v3** — Current hold-speak-release flow is fast enough. True streaming would require whisper.cpp backend and complex partial-paste UX.
