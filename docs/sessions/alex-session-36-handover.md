### `a2fc759` — Track 3 M2 architectural merge

Forge-clean `run-20260418-124215` flagged `context_menu._run_with_preload` (lines 122-208) as a near-fork of `TranscribeFileWindow._run` with browse/options/save stripped out. Subagent: *"not mechanical — needs a clear `minimal` flag and a decision on whether the preloaded variant should also respect timestamps."*

Design decisions (Alex approved option 1):

| Question | Proposed | Rationale |
|---|---|---|
| Timestamps in minimal mode | Read from `config.get("include_timestamps", False)` | Same "read from config" pattern minimal already uses for language. Tiny feature for power users — no new UI |
| Save button in minimal | Absent | Right-click transcription is the quick-read case; Copy is sufficient |
| Threading | New `blocking=False` kwarg on `.show()` | `context_menu.transcribe()` passes `blocking=True` to match prior synchronous semantics (it IS the whole program). Tray launches stay non-blocking |
| Auto-start in minimal | Yes | Matches current `_run_with_preload` behavior |

Implementation:

- `TranscribeFileWindow.__init__(self, model, config, preload_filepath=None, minimal=False)` — both kwargs default safely
- `show(self, blocking=False)` — when True, runs `_run()` directly on caller's thread
- `_run()` branches on `self._minimal` for UI construction — shares results-text + copy + close; full mode adds browse/options/transcribe/save
- `_do_transcribe(self, filepath)` — in minimal mode, lang + timestamps pulled from config; full mode pulls from UI widgets. `_transcribe_btn` access guarded with `hasattr` (only exists in full mode)
- `context_menu.transcribe()` — replaced 4-line two-step setup with single `TranscribeFileWindow(model, config, preload_filepath=filepath, minimal=True).show(blocking=True)`. Deleted the 85-line `_run_with_preload` entirely

Net: -48 lines (less than subagent's -85 estimate because we added minimal-mode branching rather than pure deletion, but gained: canonical code path, `include_timestamps` power-user knob in context-menu flow, no drift risk between the two loops).

Verified: `py_compile` clean both files; `venv/Scripts/python.exe -m unittest test_features -v` = 342/342 ok.
